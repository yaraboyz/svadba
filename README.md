# Свадебный сайт-анкета · Яков & Юлия · 12.09.2026

Статический сайт-приглашение с RSVP-анкетой. Ответы пишутся в Google-таблицу
через Google Apps Script Web App. Хостится на GitHub Pages.

## Структура

```
index.html         — сам сайт (HTML + CSS + JS в одном файле)
apps-script.gs     — код для Google Apps Script (бэкенд для записи в таблицу)
villa.webp         — фото виллы
assets/            — спрайты собак для анимации
_reference/        — исходники-референсы, на сайт не идут
```

## Что нужно сделать один раз перед запуском

### 1. Google-таблица

1. Открой [sheets.new](https://sheets.new) — создастся пустая таблица.
2. Назови её, например, «Свадьба — анкеты гостей».
3. Из адресной строки скопируй **ID таблицы** — это длинная строка
   между `/d/` и `/edit`:
   `docs.google.com/spreadsheets/d/`**`1aB2cD…XyZ`**`/edit`

### 2. Apps Script

1. В этой же таблице: **Расширения → Apps Script**.
2. В открывшемся редакторе удали всё содержимое `Code.gs` и вставь
   содержимое файла [`apps-script.gs`](./apps-script.gs).
3. В первой строке вместо `PASTE_YOUR_SHEET_ID_HERE` подставь ID из шага 1.
4. Сохрани (Ctrl/Cmd + S). Дай проекту любое имя.
5. **Развернуть → Создать развёртывание** (Deploy → New deployment).
   - Тип: **Веб-приложение** (Web app)
   - Описание: любое
   - Выполнять от имени: **Я** (Execute as: Me)
   - Доступ: **Все** (Who has access: Anyone)
   - Нажми «Развернуть». Гугл попросит дать разрешения — соглашаемся
     (он предупредит «приложение не проверено» — это нормально для
     личных скриптов, нажми «Дополнительно → Перейти к проекту»).
6. Скопируй полученный **URL веб-приложения** — он выглядит так:
   `https://script.google.com/macros/s/AKfyc…/exec`

> Если потом понадобится менять `apps-script.gs`, нужно делать **Развернуть → Управлять развёртываниями → ✏️ → Новая версия**, иначе URL будет отдавать старую версию.

### 3. Прописать URL в сайте

1. Открой `index.html`, найди строку:
   ```js
   const SCRIPT_URL = '';
   ```
2. Вставь между кавычками URL из шага 2.6.

### 4. Проверка локально

Открой `index.html` в браузере, заполни форму, отправь.
В таблице должна появиться строка с ответами. Если нет — открой DevTools (F12),
смотри вкладку Console: там будет ошибка с понятным текстом.

### 5. Деплой на GitHub Pages

1. Создай репозиторий на GitHub (публичный или приватный — Pages работает с обоими на платных тарифах; на бесплатном — публичный).
2. Из папки проекта:
   ```sh
   cd /Users/yara/projects/svadba
   git init
   git add index.html apps-script.gs README.md assets villa.webp
   git commit -m "Свадебный сайт-анкета"
   git branch -M main
   git remote add origin git@github.com:<твой-логин>/<имя-репо>.git
   git push -u origin main
   ```
3. На GitHub: **Settings → Pages → Build and deployment**
   - Source: `Deploy from a branch`
   - Branch: `main`, папка `/ (root)` → Save.
4. Через пару минут сайт будет доступен по адресу
   `https://<твой-логин>.github.io/<имя-репо>/`

Можно прикрутить свой домен — там же в Settings → Pages → Custom domain.

## Поля анкеты

| Колонка в таблице             | Откуда                                   |
|-------------------------------|------------------------------------------|
| Время                         | автоматически, момент отправки           |
| ФИО                           | поле «Как вас зовут?»                    |
| Будет ли                      | `yes` / `no`                             |
| Сколько дней                  | `full` (4 дня) / `part` (часть) / `day` (только 12-го) |
| Даты (если часть)             | свободный текст, заполняется при `part`  |
| Жильё                         | `yes` (заселить) / `self` (сам(а))       |
| Аллергия                      | `no` / `yes`                             |
| Подробности аллергии          | свободный текст при `yes`                |
| Алкоголь                      | список выбранных тегов через запятую     |
| Подпись (клятва о собаках)    | свободный текст, обязательное            |
| User-Agent                    | для отладки спама/багов                  |

## Замечания по безопасности

- Apps Script публикуется с доступом «для всех» — это нужно, чтобы анонимный
  гость мог отправить форму. URL — это секрет «через незнание», его никто не
  узнает кроме посетителей сайта. Этого достаточно для свадьбы; для серьёзных
  систем такой подход не годится.
- Если начнёт спамить — добавь в `doPost` простую защиту (длина ФИО,
  honeypot-поле, или сверка реферера `e.parameter` / `e.headers`).

## Что осталось не настроено в `index.html`

Перед запуском проверь и подправь под реальные данные:

- Номера телефонов в блоке «Контакты» (сейчас `+7 999 123 45 67/68` — фейк).
- Telegram-хендлы в той же секции (`t.me/yakov`, `t.me/yulia`).
- Тексты расписания, описание виллы, памятка «что взять» — если что-то
  не совпадает с реальностью.
